# SlaySP2Manager 前端线框与 Design Token 草案

## 1. 目标

这份文档定义前端的：

- 页面线框结构
- 主要组件关系
- 视觉节奏
- Design Token 初稿

目标不是先做高保真视觉稿，而是先把界面骨架和设计语言固定下来。

## 2. 整体视觉方向

界面要体现：

- 简洁
- 克制
- 精确
- 有呼吸感

不是：

- 游戏启动器
- 管理后台
- 论坛工具

推荐气质：

- 像高质量桌面工具
- 有一点收藏品和工作台的感觉
- 稳定、安静，但不呆板

## 3. 整体布局

建议采用桌面端三段式骨架：

```text
+-----------------------------------------------------------+
| 顶部状态条                                                 |
+------------------+----------------------------------------+
| 左侧导航           | 主内容区                               |
|                   |                                        |
|                   |                                        |
|                   |                                        |
+------------------+----------------------------------------+
```

### 3.1 顶部状态条

用于展示：

- 当前游戏目录状态
- 当前 Profile
- 更新状态
- 全局搜索入口
- 主动作入口

高度建议：

- 56px

### 3.2 左侧导航

一级导航固定为：

- 模组库
- 发现
- 配置方案
- 存档
- 设置

宽度建议：

- 216px

### 3.3 主内容区

内容区采用：

- 页面头部
- 内容主体
- 右侧详情面板或操作栏

不要每页都做满屏卡片。

## 6. 状态标签规范

### 6.1 Mod 状态

- 已启用
- 已禁用
- 可更新
- 有冲突
- 已损坏
- 未安装

### 6.2 风险状态

- 安全
- 将覆盖
- 已创建备份
- 文件占用中
- 需要确认

要求：

- 状态标签不要花哨
- 用微弱底色和细边框即可
- 不要所有状态都用高饱和色

## 7. 交互原则

### 7.1 主动作原则

每个页面只有一个视觉主按钮：

- 模组库：`更新全部`
- 发现页：`安装`
- 方案页：`应用方案`
- 存档页：`复制`

### 7.2 危险动作原则

危险动作必须：

- 明确展示将发生什么
- 展示会影响到哪里
- 展示是否已备份
- 要求二次确认

### 7.3 动效原则

只做这些动效：

- 页面进入淡入
- 列表状态切换
- 右侧详情面板滑入
- 按钮点击反馈

不做：

- 大面积弹跳
- 无意义悬浮动画
- 复杂背景动效

## 8. Design Token 初稿

## 8.1 色彩

推荐一套偏暖浅色方案。

```css
:root {
  --bg-app: #f5f1e8;
  --bg-panel: #fbf8f2;
  --bg-elevated: #ffffff;
  --bg-muted: #eee7da;

  --text-primary: #1f1d19;
  --text-secondary: #5f5a52;
  --text-muted: #8a847b;

  --border-subtle: #ddd4c6;
  --border-strong: #c6bbab;

  --accent-primary: #1e4f6f;
  --accent-primary-hover: #173e58;
  --accent-soft: #dbe7ee;

  --success: #2f6b45;
  --success-soft: #dcebdd;

  --warning: #8a5a18;
  --warning-soft: #f2e5c9;

  --danger: #8a2f2f;
  --danger-soft: #f1dddd;
}
```

说明：

- 主背景偏暖，避免纯白太硬
- 强调色选深蓝，冷静、克制
- 危险和警告色只在必要时出现

## 8.2 字体

建议：

- 中文：`HarmonyOS Sans SC`, `Source Han Sans SC`, sans-serif
- 英文：`SF Pro Display`, `IBM Plex Sans`, sans-serif

字重建议：

- 标题：600
- 分区标题：600
- 正文：400
- 辅助信息：400

## 8.3 字号

```css
:root {
  --font-size-12: 12px;
  --font-size-13: 13px;
  --font-size-14: 14px;
  --font-size-16: 16px;
  --font-size-20: 20px;
  --font-size-24: 24px;
  --font-size-32: 32px;

```

推荐使用：

- 页面标题：24px
- 区块标题：16px
- 正文：14px
- 辅助信息：13px

```css
:root {
  --space-4: 4px;
  --space-8: 8px;
  --space-12: 12px;
  --space-16: 16px;
  --space-20: 20px;
  --space-24: 24px;
  --space-32: 32px;
  --space-40: 40px;

```

页面建议：

- 页面边距：24px
- 区块间距：24px
- 卡片内边距：16px 或 20px

## 8.5 圆角

```css
:root {
  --radius-sm: 8px;
  --radius-md: 12px;
  --radius-lg: 16px;
  --radius-xl: 20px;
}
```

建议：

- 按钮：12px
- 面板：16px
- 对话框：20px

## 8.6 阴影

阴影要轻。

```css
:root {
  --shadow-soft: 0 6px 20px rgba(31, 29, 25, 0.06);
  --shadow-dialog: 0 20px 60px rgba(31, 29, 25, 0.12);
}
```

原则：

- 常规组件主要靠边框，不靠重阴影
- 只有对话框和悬浮层使用明显一点的阴影

## 8.7 动画

```css
:root {
  --duration-fast: 120ms;
  --duration-normal: 180ms;
  --duration-slow: 260ms;

  --ease-standard: cubic-bezier(0.2, 0, 0, 1);
}
```

## 9. 空状态设计

空状态不要做得像错误页。

推荐写法：

- `还没有安装任何 Mod`
- `连接 Nexus 后即可搜索和安装`
- `当前没有可更新项目`

动作按钮只给一个主动作。

## 10. 错误状态设计

错误提示分两层：

- 第一层给用户可理解结论
- 第二层可展开技术详情

示例：

- `未找到游戏目录`
- `请选择包含 SlayTheSpire2.exe 的文件夹`
- `查看详细错误`

## 11. 前端实施顺序

建议顺序：

1. 建立 `AppShell`
2. 完成左侧导航和顶部状态条
3. 完成模组库首页
4. 完成发现页
5. 完成存档页
6. 完成方案页
7. 完成设置页
8. 最后统一收敛视觉 token 和动效

## 12. V1 最重要的界面目标

V1 不追求页面多，而追求以下体验：

- 一打开就能知道当前环境状态
- 一搜索就能决定装不装
- 一更新就有安全感
- 一切换玩法就不会弄丢进度

如果这四件事做顺了，界面就已经成立。
